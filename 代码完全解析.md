## 2. 核心概念解释

### 什么是Embedding（向量化）
把文字变成一串数字（向量）。例如："手机配对" 变成 [0.2, 0.8, 0.1, ..., 共384个数字]。语义相近的文字，向量也相近。

### 什么是FAISS
Facebook的向量检索库，能在百万级向量中快速找到最相似的几个，速度毫秒级。

### 什么是RAG
检索增强生成。先检索相关文档，再让AI照着文档回答，减少幻觉。

### 什么是Temperature
控制AI输出的随机性。0=确定，1=随机。客服用0.3，创意写作用0.8。

## 3. 核心模块详解

### 3.1 src/vector_rag.py（语义检索）

**核心功能**：把用户问题转换为向量，在知识库中找最相关的文档。

**关键函数**：

**1. `__init__()` - 初始化**
```python
self.embedding_model = SentenceTransformer(模型名)
```
加载Embedding模型，首次会下载118MB。

**2. `_split_document()` - 文档切分**
- 按段落切分文档
- 过滤太短的（<20字）
- 切分太长的（>500字）

**为什么**：整篇文档太长，切成小段落更精准。

**3. `_build_index()` - 构建向量索引**
```python
# 1. 把所有段落转换为向量
embeddings = self.embedding_model.encode(all_chunks)

# 2. 创建FAISS索引
self.index = faiss.IndexFlatIP(维度)
self.index.add(embeddings)
```

**4. `retrieve()` - 检索**
```python
# 1. 把问题转换为向量
query_embedding = self.embedding_model.encode([query])

# 2. 在FAISS中搜索最相似的top_k个
scores, indices = self.index.search(query_embedding, top_k)

# 3. 返回文档
return [(文档名, 内容, 相似度), ...]
```

**这就是语义检索的核心**：问题和文档都变成向量，用向量相似度找最相关的文档。

---

### 3.2 src/llm_client.py（LLM调用）

**核心功能**：封装大模型API调用，处理重试、统计tokens。

**关键函数**：

**1. `_call_api_with_retry()` - API调用与重试**
```python
for attempt in range(max_retries):
    try:
        response = self.client.chat.completions.create(...)
        return answer, input_tokens, output_tokens
    except Exception as e:
        if "429" in str(e):  # 限流错误
            wait_time = (attempt + 1) * 3  # 3秒、6秒、9秒
            time.sleep(wait_time)
            continue
        raise e
```

**为什么重要**：API有限流，自动重试提高成功率。

**2. `answer_question()` - 生成回答**
```python
# 1. 把检索到的文档拼成上下文
context = "\n\n".join([f"[文档{i}]{content}" for ...])

# 2. 填充prompt模板
user_prompt = template.format(context=context, question=question)

# 3. 调用API
messages = [
    {"role": "system", "content": system_prompt},
    {"role": "user", "content": user_prompt}
]
return self._call_api_with_retry(messages)
```

**这就是RAG**：检索文档 → 拼成prompt → 发给LLM → 生成回答。

### 3.3 src/hallucination_detector.py（幻觉检测）

**核心功能**：检测AI回答中可能编造的内容。

**方法**：
- 提取AI回答的关键词
- 检查这些关键词有多少能在原文档中找到
- 支撑率 = 有支撑的关键词数 / 总关键词数

**关键函数**：

**1. `extract_keywords()` - 提取关键词**
```python
keywords = set()

# 提取"数字+单位"（如7天、5ATM）
for pattern in number_patterns:
    matches = re.findall(pattern, text)
    keywords.update(matches)

# 移除虚词
clean_text = text.replace('的', ' ').replace('了', ' ')

# 提取2-5字的中文短语
phrases = re.findall(r'[一-龥]{2,5}', clean_text)
keywords.update(phrases)

return keywords
```

**2. `detect()` - 幻觉检测**
```python
# 1. 提取回答和文档的关键词
answer_keywords = self.extract_keywords(answer)
doc_keywords = self.extract_keywords(all_doc_content)

# 2. 计算支撑率
supported = answer_keywords & doc_keywords  # 交集
score = len(supported) / len(answer_keywords)

# 3. 判断风险
if score >= 0.7:
    risk_level = "low"
elif score >= 0.5:
    risk_level = "medium"
else:
    risk_level = "high"
```

**简历亮点**：不需要标准答案，自动检测幻觉，方法简单有效。

### 3.4 src/analytics.py（数据统计）

**核心功能**：记录交互日志，计算统计指标。

**关键函数**：

**1. `log_interaction()` - 记录日志**

```python
log_entry = {
    "timestamp": datetime.now().isoformat(),
    "question": question,
    "answer": answer,
    "input_tokens": input_tokens,
    "output_tokens": output_tokens,
    "cost": cost,

    f.write(json.dumps(log_entry) + '\n')
```

**2. `get_statistics()` - 计算指标**
```python
total_cost = sum(log["cost"] for log in logs)
useful_count = sum(1 for log in logs if log["user_feedback"] == "useful")
adoption_rate = useful_count / feedback_count

return {
    "total_interactions": total,
    "total_cost": total_cost,
    "adoption_rate": adoption_rate,
    ...
}
```

**这是数据驱动决策的基础**：记录完整数据 → 计算指标 → 指导迭代。

## 4. 页面文件详解

**核心流程**：

```python
# 1. 初始化

# 2. 用户输入
```

## 5. 代码协作关系

### 5.1 用户提问的完整流程

```
用户点击"提问"按钮
    ↓
main.py 接收问题
    ↓
调用 vector_rag.retrieve(question)
    ↓
vector_rag.py:
  - 调用 embedding_model.encode() 把问题变成向量
  - 调用 faiss.search() 检索最相似的文档
  - 返回 [(文档名, 内容, 相似度), ...]
    ↓
main.py 调用 llm_client.answer_question(question, docs)
    ↓
llm_client.py:
  - 拼接文档和问题成prompt
  - 调用 OpenAI API
  - 返回 回答和tokens统计
    ↓
main.py 调用 hallucination_detector.detect(answer, docs)
    ↓
hallucination_detector.py:
  - 提取关键词
  - 计算支撑率
  - 返回 {"score": 0.75, "risk_level": "low"}
    ↓
main.py 调用 analytics.log_interaction(...)
    ↓
analytics.py:
  - 写入 data/logs/interactions.jsonl
    ↓
main.py 显示结果
```

## 6. 常见编程概念解释

### 什么是类（class）
类是一个"工具箱"，把相关的函数和数据放在一起。`VectorRAG`类包含检索相关的所有功能。

### self是什么
`self`代表类的实例本身。`self.model`就是"这个对象的model属性"。

### 下划线前缀（_function）
表示"私有方法"，只在类内部使用，外部不应该调用。

### try...except
异常处理。`try`块里的代码如果出错，就跳到`except`块。

### @st.cache_resource
让Streamlit缓存函数返回值，避免重复加载模型。
